# Wazuh-MISP-IP-Address-Enrichment-Integration


## Overview

This project provides a custom integration between **Wazuh** and **MISP (Malware Information Sharing Platform)** to enrich alerts containing IP addresses with threat intelligence from MISP.

It is based on the official Wazuh–MISP file hash integration:

https://github.com/MISP/wazuh-integration

The original integration correlates file hashes generated by FIM (syscheck).  
This implementation extends the same architecture and logic to support IP address indicators.

---

## Features

- Enrich Wazuh alerts using MISP threat intelligence
- Supports IPv4 and IPv6 indicators
- Queries MISP via REST API (`attributes/restSearch`)
- Generates new Wazuh alerts on matches
- Optional automatic sightings submission to MISP
- Compatible with private or self-hosted MISP deployments
- Lightweight and dependency-free (uses Wazuh framework Python)


---

## Requirements

- Wazuh Manager 4.x
- MISP instance with API access enabled
- Valid MISP API key with attribute search permissions
- Network connectivity between Wazuh Manager and MISP
- Python `requests` library (included in Wazuh framework)

## Installation

### 1. Copy the Integration Script

Place the script in the Wazuh integrations directory:
/var/ossec/integrations/custom-misp-ips.py

### 2. Set correct permissions:


```bash
chmod 750 /var/ossec/integrations/custom-misp-ips.py
chown root:wazuh /var/ossec/integrations/custom-misp-ips.py
```
### 3. Configure Wazuh Integration

Edit: ``` /var/ossec/etc/ossec.conf ```
Add the following block:
```bash
<integration>
  <name>custom-misp-ips.py</name>
  <hook_url>https://MISP_HOST</hook_url>
  <api_key>MISP_API_KEY</api_key>
  <group>syslog</group>
  <alert_format>json</alert_format>
  <options>{
      "timeout": 10,
      "retries": 3,
      "debug": false,
      "tags": ["tlp:white", "tlp:clear", "malware"],
      "push_sightings": true,
      "sightings_source": "wazuh"
  }</options>
</integration>
```

**Configuration Parameters:**

| Parameter | Description | Example |
|-----------|-------------|---------|
| `<name>` | Integration script filename | `custom-misp-ips.py` |
| `<hook_url>` | MISP instance URL | `https://misp.example.com` |
| `<api_key>` | MISP API authentication key | Your MISP AuthKey |
| `<group>` | Rule groups to trigger integration | `syslog,fortigate,firewall` |
| `<alert_format>` | Alert format sent to script | `json` |
| `<options>` | Additional configuration (JSON) | See below |

### 4. Add Custom Detection Rules:
Edit:   ```/var/ossec/etc/rules/local_rules.xml```
```bash
<group name="misp,threat-intel,">

  <rule id="101806" level="0">
    <decoded_as>json</decoded_as>
    <description>MISP: IP check</description>
    <field name="integration">misp_fortigate_ips</field>
    <options>no_full_log</options>
  </rule>

  <rule id="101807" level="0">
    <if_sid>101806</if_sid>
    <field name="misp_fortigate_ips.found">0</field>
    <description>MISP: IP not found</description>
  </rule>

  <rule id="101808" level="12">
    <if_sid>101806</if_sid>
    <field name="misp_fortigate_ips.found">1</field>
    <description>MISP: IP matched</description>
  </rule>

  <rule id="101809" level="10">
    <if_sid>101806</if_sid>
    <field name="misp_fortigate_ips.error">403</field>
    <description>MISP ERROR: Invalid MISP credentials, check api_key (AuthKey)</description>
  </rule>

  <rule id="101810" level="10">
    <if_sid>101806</if_sid>
    <field name="misp_fortigate_ips.error">429</field>
    <description>MISP ERROR: Rate limit exceeded, too many requests</description>
  </rule>

  <rule id="101811" level="10">
    <if_sid>101806</if_sid>
    <field name="misp_fortigate_ips.error">500</field>
    <description>MISP ERROR: $(misp_fortigate_ips.description)</description>
  </rule>


</group>
```
### 5. Restart Wazuh Manager:

**For systemd installations:**
```bash
systemctl restart wazuh-manager
```

**For Docker installations:**
```bash
docker compose restart wazuh.manager
```
